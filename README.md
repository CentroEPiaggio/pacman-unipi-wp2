    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-unipi-grasp-datasets" class="anchor" href="#unipi-grasp-datasets" aria-hidden="true"><span class="octicon octicon-link"></span></a>unipi-grasp-datasets</h1>

<p>This pacakge contains experiment protocols on how to generate grasp datasets. Everything is based on <a href="http://wiki.ros.org/indigo/Installation/Ubuntu">ROS indigo</a>/<a href="http://releases.ubuntu.com/14.04/">Ubunt 14.04</a>. You must have a <a href="http://wiki.ros.org/catkin/Tutorials/create_a_workspace">catkin workspace</a> configured in order to follow the instructions.</p>

<h2>
<a id="user-content-equipment--dependencies" class="anchor" href="#equipment--dependencies" aria-hidden="true"><span class="octicon octicon-link"></span></a>Equipment &amp; Dependencies</h2>

<ul class="task-list">
<li><a href="https://github.com/CentroEPiaggio/phase-space">PhaseSpace</a></li>
<li>
<a href="http://wiki.ros.org/openni2_launch">Asus/Kinect</a> (install from synaptic/apt-get recommended)</li>
<li><a href="https://github.com/CentroEPiaggio/force-torque-sensor">F/T</a></li>
<li><a href="https://github.com/CentroEPiaggio/intrinsic-tactile-toolbox">Intrinsic Tactile Toolbox</a></li>
<li><a href="https://github.com/CentroEPiaggio/pisa-iit-soft-hand">SoftHand</a></li>
<li><a href="https://github.com/CentroEPiaggio/flexiforce-glove">Flexiforce Glove</a></li>
<li><a href="https://github.com/CentroEPiaggio/calibration">Calibration</a></li>
<li><a href="https://github.com/pacman-project/pacman-object-database">PaCManObjectDatabse</a></li>
</ul>

<p>You need to install them all.</p>

<h2>
<a id="user-content-guidelines-to-create-grasp-experiment-scenarios" class="anchor" href="#guidelines-to-create-grasp-experiment-scenarios" aria-hidden="true"><span class="octicon octicon-link"></span></a>Guidelines to create grasp experiment scenarios</h2>

<ul class="task-list">
<li>If you use additional equipments or dependencies, please, add the link where to find them in the list above</li>
<li>Create a package of your scenario using <code>catkin_create_pkg &lt;scnarioname&gt;</code> (more arguments can be useful to fill the <code>package.xml</code>)</li>
<li>Follow the template of <code>scenario1</code>, create at least these files:

<ul class="task-list">
<li>
<code>launch/uploadSetup.launch</code> to bring up all devices</li>
<li>
<code>data/startRecording.sh</code> to define your recording protocol (if possible, record only sensor data and parameters)</li>
<li>
<code>launch/playExperiment.launch</code> to play back recorded data</li>
</ul>
</li>
<li>IMPORTANT: Do not commit your bag files to this repository! One alternative is to upload your bag files to the <a href="http://131.114.31.70/">CentroPiaggio server</a> under the <code>datasets</code> folder as a single zip, share the file and provide a public link. If you don't have an account, ask to the <a href="http://www.centropiaggio.unipi.it/administrative-staff.html">technical dept.</a> or your supervisor.</li>
<li>EVEN MORE IMPORTANT: Document your scenario in this README file with at least two sections:

<ul class="task-list">
<li>
<code>Experiment protocol</code> to explain how the experiments must be ran/were ran</li>
<li>
<code>How to play the recorded data</code> to describe how to play the data you obtained and provide download links to your bag files.</li>
</ul>
</li>
<li>The package <code>sync_data</code> allow you to synchronize your data either for recording (not recommended) or for data being played. To this end, you should:

<ul class="task-list">
<li>Create <code>msg/&lt;scenarioname&gt;.msg</code> file to define which type of message you want to synchronize</li>
<li>Write a <code>src/&lt;scnarioname&gt;sync.cpp</code> node to subscribe to the topics you want to synchronize</li>
<li>Create a <code>launch/&lt;scenarioname&gt;.launch</code> file to facilitate the use</li>
</ul>
</li>
</ul>

<h2>
<a id="user-content-scenarios-and-datasets" class="anchor" href="#scenarios-and-datasets" aria-hidden="true"><span class="octicon octicon-link"></span></a>Scenarios and Datasets</h2>

<h3>
<a id="user-content-scenario-1-phasespaceasussofthandflexiforceglove" class="anchor" href="#scenario-1-phasespaceasussofthandflexiforceglove" aria-hidden="true"><span class="octicon octicon-link"></span></a>Scenario 1: <a href="http://www.phasespace.com/">PhaseSpace</a>/<a href="http://www.asus.com/Multimedia/Xtion_PRO_LIVE/">Asus</a>/<a href="http://www.qbrobotics.com/#!softhand/c1njg">SoftHand</a>/<a href="https://github.com/CentroEPiaggio/flexiforce-glove">FlexiforceGlove</a>
</h3>

<p>This scenario has been designed for WP2 in the <a href="http://www.pacman-project.eu/">PaCMan</a> project.</p>

<h4>
<a id="user-content-experiment-protocol" class="anchor" href="#experiment-protocol" aria-hidden="true"><span class="octicon octicon-link"></span></a>Experiment protocol</h4>

<ol class="task-list">
<li><p>Connect all devices, use the links above to go directly to their instructions within.</p></li>
<li><p>Change the working directiory, open and work on the data directory <code>roscd scenario1/data</code>.</p></li>
<li><p>Calibrate the asus/phasespace sytems. instructions are in the <a href="https://github.com/CentroEPiaggio/calibration/blob/master/README.md">README</a> of the <a href="https://github.com/CentroEPiaggio/calibration">calibration</a> package.</p></li>
<li><p>Select an object from the <a href="https://github.com/pacman-project/pacman-object-database">PaCMan Object Databse</a> that is physically available and is included in your object pose estimation software.</p></li>
<li><p>Calibrate a knwon tracker with the object, instructions are in the <a href="https://github.com/CentroEPiaggio/calibration/blob/master/README.md">README</a> of the <a href="https://github.com/CentroEPiaggio/calibration">calibration</a> package.</p></li>
<li><p>Upload the setup for the experiment, open a different terminal and type <code>roslaunch scenario1 uploadSetup.launch</code>, and go back to initial terminal. A window should open that allow you to see the processing of the data online.</p></li>
<li>
<p>Perform and record grasp actions. The subjects are advised about the recording protocol. The actions are</p>

<ul class="task-list">
<li>Place the hand at rest position</li>
<li>
<code>./startRecording.sh</code> (please, follow instructions on the terminal and wait for the GO! signal)</li>
<li>Grasp the object and lift after the <code>GO!</code> signal</li>
<li>Wait 2 seconds</li>
<li>Replace the object on the table</li>
<li>Go to the same rest position and wait until recording process is stopped automatically</li>
</ul>
</li>
<li><p>If you want to try a different grasp on the same object, go to point 6. If you want to change the object, go the second terminal you opened, kill that process, go to point 23</p></li>
</ol>

<p>Notes on the recorded data:</p>

<ul class="task-list">
<li>Recall only raw sensor data is recorded in the bag files at their max frequency. </li>
<li>Recall that parameter for each grasp is saved to disk prior the recording to replicate the data processing. </li>
<li>Point clouds are recorded using 20 of 21 messages at 30Hz, if you want to change this, modify the <a href="/CentroEPiaggio/unipi-grasp-datasets/blob/master/scenario1/launch/uploadSetup.launch"><code>uploadSetup.launch</code></a>.</li>
<li>
</li>
</ul>

<h4>
<a id="user-content-how-to-play-the-recorded-data" class="anchor" href="#how-to-play-the-recorded-data" aria-hidden="true"><span class="octicon octicon-link"></span></a>How to play the recorded data</h4>

<ol class="task-list">
<li><p>Download the <a href="http://131.114.31.70:8080/share.cgi?ssid=096EZd5&amp;fid=096EZd5&amp;ep=LS0tLQ==">file</a> containing all bag files and uncompress it in the <code>data</code> folder.</p></li>
<li><p>Type <code>roslaunch scenario1 playExperiment.launch experiment_name:=NAME</code>, i.e. the folder name. This already loads the parameters used during the experiment.</p></li>
</ol>

<p>If you want to have a specific information being published in a(several) topic(s) in a given point in time, you can <code>ros topic YOUR_TOPIC</code> and hit <code>space</code> to pause the playback at the desired moment, and hit <code>s</code> to perform a tiny step on the playback. Check the terminal where you echoed the topic and take the required infromation, for instace, this procedure can be used to have a static grasp captured from the recording.</p>
